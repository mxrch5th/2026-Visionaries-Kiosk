# 얼굴 데이터 정리 방법

이 프로젝트는 `50대 이상이면 쉬운모드`, `50대 미만이면 일반 화면`으로 나누는 이진 분류 모델을 사용합니다.

## 폴더 구조

```text
dataset/
  train/
    under50/
    over50/
  val/
    under50/
    over50/
```

- `under50`: 50대 미만 얼굴 이미지
- `over50`: 50대 이상 얼굴 이미지
- `train`: 학습용 이미지
- `val`: 검증용 이미지

## 팀원별 데이터 모으기

각자 70장씩 모을 경우, 처음에는 아래처럼 팀원별 폴더로 보관해도 됩니다.

```text
team_data/
  member_01/
    under50/
    over50/
  member_02/
    under50/
    over50/
```

학습 전에는 모든 팀원 데이터를 `dataset/train`과 `dataset/val`로 합치면 됩니다. 보통 전체 이미지의 80%는 `train`, 20%는 `val`에 넣습니다.

## 주의할 점

- 같은 사람 사진이 `train`과 `val`에 동시에 들어가지 않게 하세요.
- 얼굴이 너무 작거나 심하게 가려진 이미지는 제외하세요.
- 개인정보 보호를 위해 원본 얼굴 이미지는 GitHub에 올리지 않는 것을 권장합니다.
- 데이터가 한쪽으로 치우치면 모델도 치우칩니다. `under50`과 `over50` 수량을 최대한 비슷하게 맞추세요.
